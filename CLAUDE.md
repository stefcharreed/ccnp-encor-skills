# CLAUDE.md — ccnp-encor-skills

Public CCNP ENCOR study skill catalog. Repo: https://github.com/stefcharreed/ccnp-encor-skills (public).

## What this is
Phase 1 (this repo, public): collecting CCNP ENCOR topics as individual
Claude Code skills under `1.0.0/skills/<topic>/SKILL.md`. Each skill is
generated by the `ccnp-encor:ccnp-note` plugin skill from study notes (raw
notes, IOS-XE config, `show` output, or spoken description).

Phase 2 (separate private repo,
[network-troubleshooting-agent](https://github.com/stefcharreed/network-troubleshooting-agent),
**planning underway, no agent code written yet**): combine the topic skills
from this catalog into a single agent capable of diagnosing real network
problems across multiple topic areas, not just one isolated skill at a time.
That repo already has architecture notes (retrieval-not-reasoning design,
a worked InterfaceDown+config-drift correlation example) and is gated on this
catalog reaching Tier 1 coverage first — see its `ARCHITECTURE-NOTES.md` for
the current tiering (FHRP, DHCP, NAT, ACLs, first-hop security, AAA next;
SNMP/syslog/NetFlow/IP SLA after that; QoS/Wireless/Automation/Architecture
deferred). Kept in a separate private repo since it will likely involve real
device interaction patterns. The public README teases this with a CTA to
reach out via LinkedIn — do not start phase 2 *implementation* work in this
repo, and do not restructure this repo's layout for portability unless
explicitly asked.

## Layout
- `1.0.0/skills/<TOPIC>/SKILL.md` — one file per CCNP topic
- `1.0.0/skills/ccnp-note/SKILL.md` — the skill that generates new topic skills
- `1.0.0/package.json` — plugin manifest
- `1.0.0/.in_use/` — runtime PID lock file, gitignored, never commit

## Workflow rule — always push after adding a skill
**Every time a new topic skill is written or an existing one is edited,
commit and push immediately.** Do not leave new skills uncommitted.

```bash
cd ~/.claude/plugins/cache/local/ccnp-encor
git add -A
git commit -m "Add <topic> skill"   # or "Update <topic> skill"
git push
```

This applies whether the skill was created via `/ccnp-note <topic>` or
edited directly. No need to ask for confirmation before this push — it's a
private repo dedicated to this catalog, committing/pushing study notes here
is pre-authorized.

## Safety
- This repo is public — do not commit anything that shouldn't be public
  (real device state, credentials, personal info beyond what's already in
  the README). Phase 2 (device interaction, lab details) belongs in the
  private `network-troubleshooting-agent` repo instead.
- Never commit `.in_use/` or other runtime state.
