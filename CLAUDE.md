# CLAUDE.md — ccnp-encor-skills

Private CCNP ENCOR study skill catalog. Repo: https://github.com/stefcharreed/ccnp-encor-skills (private).

## What this is
Phase 1: collecting CCNP ENCOR topics as individual Claude Code skills under
`1.0.0/skills/<topic>/SKILL.md`. Each skill is generated by the
`ccnp-encor:ccnp-note` plugin skill from study notes (raw notes, IOS-XE
config, `show` output, or spoken description).

Phase 2 (not started): restructure for export/import into other agents —
likely flatten `1.0.0/skills/<topic>` to a plain `skills/<topic>/SKILL.md`
repo layout so it can be cloned/symlinked without the plugin-cache path.
Do not do this restructuring unless explicitly asked.

## Layout
- `1.0.0/skills/<TOPIC>/SKILL.md` — one file per CCNP topic
- `1.0.0/skills/ccnp-note/SKILL.md` — the skill that generates new topic skills
- `1.0.0/package.json` — plugin manifest
- `1.0.0/.in_use/` — runtime PID lock file, gitignored, never commit

## Workflow rule — always push after adding a skill
**Every time a new topic skill is written or an existing one is edited,
commit and push immediately.** Do not leave new skills uncommitted.

```bash
cd ~/.claude/plugins/cache/local/ccnp-encor
git add -A
git commit -m "Add <topic> skill"   # or "Update <topic> skill"
git push
```

This applies whether the skill was created via `/ccnp-note <topic>` or
edited directly. No need to ask for confirmation before this push — it's a
private repo dedicated to this catalog, committing/pushing study notes here
is pre-authorized.

## Safety
- Private repo — do not make it public without asking (these are personal
  study notes, not portfolio code).
- Never commit `.in_use/` or other runtime state.
